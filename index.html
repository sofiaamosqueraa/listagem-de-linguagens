Nome Definição Vantagens Desvantagens
Binário É um linguagem utilizado em computadores, ou seja, trata-se da forma de comando de softwares para que o armazenamento de dados seja feito. Tal linguagem é denominada “Código Binário” porque um computador é capaz de decodificar apenas dois algarismos: 0 ou 1. Tais que significam dois estados de energia, onde 0 indica a ausência e 1 a presença.
  • O código binário é diretamente interpretável pelo processador, resultando em maior rapidez na execução do programa.

  • A dificuldade de leitura e manipulação do código dificulta a detecção de falhas e vulnerabilidades por pessoas mal-intencionadas.
    		•
  • A linguagem binária é complexa e exige tempo e dedicação para ser aprendida e dominada. A leitura e escrita do código podem ser desafiadoras para programadores iniciantes.

  • O código binário é pouco intuitivo e difícil de ser interpretado, o que dificulta a compreensão do funcionamento do programa.
    		•
    Assembly É uma linguagem de baixo nível de programação usada para a interface com o hardware do computador. A linguagem assembly usa comandos estruturados em vez de números e permite que os programadores consigam ler mais facilmente o código.
  • Permite um controle mais preciso sobre o hardware do computador, resultando em código mais eficiente e otimizado.

  • Promove um melhor entendimento da arquitetura do computador e do funcionamento do hardware.
    		•
  • Linguagem de baixo nível com sintaxe complexa e específica para cada arquitetura de processador.
    		•
    Low code O low-code é uma abordagem de desenvolvimento de software que requer pouco ou nenhum código para construir aplicações e processos. Uma plataforma de desenvolvimento de low-code utiliza interfaces visuais com funcionalidades simples de lógica de arrastar e largar em vez de extensas linguagens de programação.
  • Permite a criação de aplicações complexas com menos tempo e esforço, utilizando interfaces visuais e ferramentas de arrastar e soltar.
  • Permite que pessoas sem conhecimento de programação tradicional criem aplicações funcionais, expandindo as possibilidades de criação e inovação.
    		•
  • As plataformas low-code oferecem um conjunto de funcionalidades pré-definidas, limitando a personalização e a adaptação a necessidades específicas.
  • As plataformas low-code geralmente exigem assinaturas ou licenças de uso, o que pode gerar custos adicionais para a empresa.
    		•
    Linguagens embarcadas Programação embarcada é o nome dado a programas direcionados a sistemas embarcados, que por sua vez, são aqueles que diferente de sistemas de uso geral, são planejados para o uso de tarefas pré-definidas de uso específico, por exemplo um roteador. São usadas linguagens de baixo-nível, em sua maioria C e Assembly.
  • Permitem a criação de código eficiente e compacto, ideal para aplicações que exigem alto desempenho e baixo consumo de energia.
  • As linguagens embarcadas oferecem acesso direto ao hardware do sistema, permitindo um controle preciso sobre seus recursos.
    		•
  • Exigem um conhecimento profundo do hardware e da linguagem para escrever código eficiente e confiável.
  • O código escrito em linguagens embarcadas geralmente não é portátil entre diferentes plataformas de hardware.
